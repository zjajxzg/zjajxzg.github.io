### 策略模式

------

#### 背景：

​	相信大家在写业务需求的时候总会遇到这样的情况，根据前端传的参数来区分不同的场景来执行不同的任务，于是就有了下面的代码：

```java
// 场景A
if (A) {
    // do some things...
} else if (B) {
    // do some things...
} else if (C) {
    // ...
}
```

当业务场景不多的情况下，这样的代码我们还能接受，但是一旦业务场景越来越复杂，可能会有10多个场景，你会发现这里会形成if...else...噩梦，这个时候就可以用策略模式来消除if-else。
